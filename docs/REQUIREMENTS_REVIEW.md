# Статическое ревью требований (Static Requirements Review)

**Проект:** VentSearch — Система подбора промышленных вентиляторов  
**Объект анализа:** Спецификации функциональных (FR) и нефункциональных требований (NFR)  
**Дата:** 24 апреля 2026 г.  
**Ответственный:** Андреев А. (Team Lead / QA-acting)  

---

## 1. Цель документа
Провести проверку требований на ранней стадии разработки (фаза Construction, итерация 1) для выявления логических противоречий, двусмысленностей и нетестируемых формулировок. Статическое тестирование проводится без запуска программного кода.

## 2. Критерии оценки (согласно лекции от 30.03)
* **Однозначность:** требование имеет только одну интерпретацию.
* **Проверяемость (Тестируемость):** наличие четкой метрики для подтверждения выполнения.
* **Непротиворечивость:** отсутствие конфликтов с бизнес-целями или другими требованиями.
* **Полнота:** достаточность информации для реализации разработчиком.

## 3. Результаты ревью функциональных требований (FR)

| ID | Исходное требование | Результат проверки | Замечания и корректировки |
| :--- | :--- | :--- | :--- |
| **F-01** | Поиск вентилятора по параметрам | **Warning** | **Неполнота.** Не указан список обязательных фильтров. <br> *Исправить на:* «Поиск по: типу, расходу (м³/ч), давлению (Па), мощности (кВт), шуму (дБ) и напряжению (В)». |
| **F-02** | Результаты поиска за 2 секунды | **Warning** | **Ошибочная классификация.** Это требование к производительности. <br> *Решение:* Перенести в раздел NFR (см. NF-01). |
| **F-06** | Единый каталог от нескольких производителей | **CRITICAL ERROR** | **Противоречие бизнес-логике.** Заказчик (ООО «ЗАВОД ВЕНТМАШ») — единственный производитель. <br> *Исправить на:* «Поиск осуществляется исключительно по каталогу продукции ООО «ЗАВОД ВЕНТМАШ». |
| **F-09** | Предложение аналогов при отсутствии точного совпадения | **Ambiguous** | **Двусмысленность.** Что считать аналогом? <br> *Исправить на:* «Предлагать модели с отклонением по давлению/расходу не более ±5% от запроса». |
| **F-11** | Интеграция с 1С | **Deleted** | **Out of Scope.** Решением Тимлида задача исключена из текущего академического проекта для обеспечения стабильности MVP. |

## 4. Результаты ревью нефункциональных требований (NFR)

| ID | Категория (ISO 25010) | Статус | Замечание / Рекомендация |
| :--- | :--- | :--- | :--- |
| **NF-01** | Производительность | **OK** | Метрика в 2 секунды однозначна и проверяема секундомером/логами API. |
| **NF-02** | Удобство (Usability) | **Warning** | **Нетестируемо.** Формулировка «понятность без обучения» субъективна. <br> *Рекомендация:* Заменить на «Время успешного подбора модели новым пользователем < 3 мин». |
| **NF-04** | Безопасность | **OK** | Требование об ограничении доступа по внешним IP четкое и критичное для локальной версии. |
| **NF-05** | Сопровождаемость | **Warning** | **Двусмысленность.** «Обновление без программиста» требует уточнения способа. <br> *Рекомендация:* «Добавление новых моделей через импорт CSV-файла в БД». |
| **NF-06** | Переносимость | **OK** | Четко указаны ОС (Windows 10/11) и отсутствие платных лицензий. |

## 5. Итоговое заключение
Требования содержали критическую ошибку в определении количества производителей (F-06), что могло привести к избыточному усложнению архитектуры БД. После внесения правок, зафиксированных в данном ревью, документация считается пригодной для разработки тест-кейсов и финальной реализации поиска.

**Статус:** Условно одобрено (требуется актуализация FR-документа).
