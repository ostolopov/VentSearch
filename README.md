Система подбора промышленных вентиляторов для дистрибьюторов B2B. Решает проблему ручного поиска по Excel и PDF-каталогам, сокращая время подбора с часов до минут.

О проекте

Промышленные вентиляторы — техническое оборудование для заводов, систем вентиляции, электростанций. В отличие от бытовых устройств, их выбор требует точного соответствия параметрам: производительность, давление, мощность, уровень шума.

Существующий процесс у дистрибьюторов: поиск по разрозненным Excel-таблицам и PDF-каталогам от разных производителей. Инженер тратит до нескольких часов на подбор одной модели.

VentSearch объединяет каталоги производителей в единую базу с многопараметрическим поиском и визуализацией характеристик.


Функциональность

Поиск по техническим параметрам: тип вентилятора, производительность, давление, мощность, уровень шума, напряжение питания.

Визуализация рабочих характеристик в виде графиков зависимости давления от производительности.

Сравнение до трех моделей на одном графике.

Экспорт результатов в PDF.

Работа без подключения к интернету — локальная установка на ПК клиента.


**База данных:** PostgreSQL с индексами B-tree для оптимизации поиска.

**Алгоритмы:** Bloom filter + B деревья для предфильтрации запросов, снижение нагрузки на БД.

**Платформа:** Windows 10+, локальная сеть.


**Логика работы VentSearch:**

**Инициализация:** При запуске docker-compose поднимается база PostgreSQL и FastAPI сервер.

**Запрос:** Пользователь вводит в веб-интерфейсе требуемый расход ($Q$) и давление ($P$).

**Поиск:** Бэкенд делает запрос к БД, фильтруя таблицу qp_curves по индексированным полям flow_rate и pressure_pa.

**Расчет:** Программа находит пересечение рабочей точки пользователя с кривыми вентиляторов из БД (используя интерполяцию точек из qp_curves).

**Ранжирование:** Результаты сортируются по КПД или цене и отдаются на фронтенд.


Установка

Требования: Windows 10 или выше, 4 ГБ RAM, 500 МБ свободного места.

Установка PostgreSQL и инициализация базы данных.

Запуск серверной части: `uvicorn backend.app:app --host 0.0.0.0 --port 8000`.

Открытие клиентской части в браузере по адресу localhost.

Для хостинга в Yandex Cloud VM с автодеплоем через GitHub Actions смотрите:
`docs/DEPLOY_VM.md`
