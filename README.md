## 🎯 О проекте

**Проблема:**  
Подбор промышленных вентиляторов дистрибьюторами сегодня – это часы ручного поиска по разрозненным Excel-таблицам и PDF-каталогам разных производителей. Инженер должен вручную сопоставлять десятки параметров (расход, давление, мощность, шум), что приводит к ошибкам и затянутым срокам.

**Решение:**  
VentSearch объединяет каталоги в единую оптимизированную базу данных и предоставляет многопараметрический поиск с визуализацией характеристик. Время подбора сокращается с нескольких часов до **1–2 минут**.

> Проект выполняется в рамках академического курса «Технологии разработки ПО» по заказу реального дистрибьютора промышленного вентиляционного оборудования – **ЗАО «ЗАВОД ВЕНТМАШ»**.

## ✨ Ключевые возможности

- 🔍 **Многопараметрический поиск**  
  Фильтрация по типу вентилятора, производительности, давлению, мощности, уровню шума, напряжению питания.

- 📈 **Интерактивные графики**  
  Визуализация кривых «давление – производительность». До трёх моделей на одном графике для сравнения.

- 🧮 **Умное ранжирование**  
  Автоматический расчёт пересечения рабочей точки пользователя с характеристиками вентилятора, сортировка по КПД или цене.

- 📄 **Экспорт в PDF**  
  Формирование отчёта с выбранными моделями и их параметрами.

- 🖥️ **Локальная работа без интернета**  
  Вся система устанавливается на ПК заказчика (Windows 10/11), обмен данными только внутри Docker-контейнеров.

- ⚡ **Высокая производительность**  
  Индексы B-tree, предфильтрация Bloom-фильтром, снижение нагрузки на БД.

## 🧱 Архитектура и стек

| Компонент          | Технология                                         |
| ------------------ | -------------------------------------------------- |
| Бэкенд             | Python 3.10+, FastAPI, Uvicorn                     |
| База данных        | PostgreSQL 15+ (индексы B-tree)                    |
| Предфильтрация     | Алгоритм Bloom filter + B-деревья                  |
| Контейнеризация    | Docker, Docker Compose                             |
| Фронтенд           | Веб-интерфейс (HTML/CSS/JavaScript, локальный хост)|
| Платформа          | Windows 10+, Linux (серверная часть по желанию)    |

### 🔄 Принцип работы

1. **Инициализация** – `docker-compose up` поднимает PostgreSQL и FastAPI сервер.
2. **Запрос** – пользователь задаёт в веб-интерфейсе требуемые расход `Q` и давление `P`.
3. **Поиск** – бэкенд запрашивает таблицу `qp_curves`, фильтруя по индексированным полям `flow_rate` и `pressure_pa`.
4. **Расчёт** – интерполяция точек кривых, нахождение ближайших моделей, пересекающих рабочую точку.
5. **Ранжирование** – сортировка по КПД или цене, выдача результата на фронтенд.

## 🚀 Быстрый старт

> 💡 **Для обычного пользователя (дистрибьютора) подготовлена пошаговая инструкция с пояснениями и скриншотами.**  
> См. полную версию в файле **[docs/INSTALL_LOCAL.md](./docs/INSTALL_LOCAL.md)**.

### 📋 Что потребуется

- Windows 10 или 11
- Docker Desktop (установка займёт 5 минут)
- 4 ГБ ОЗУ, 500 МБ свободного места на диске

### ⚡ Запуск в три шага

1. **Скачайте проект**  
   Перейдите на [github.com/ostolopov/VentSearch](https://github.com/ostolopov/VentSearch), нажмите **Code → Download ZIP** и распакуйте архив в удобную папку.

2. **Запустите `run.bat`**  
   Дважды кликните по файлу `run.bat` в папке проекта. При первом запуске Docker скачает необходимые образы (до 5 минут).  
   *Не закрывайте появившееся консольное окно — это работающий сервер.*

3. **Откройте программу**  
   В браузере перейдите по адресу: **[http://127.0.0.1:8000](http://127.0.0.1:8000)**  
   Готово! Вы в веб-интерфейсе VentSearch.
